# Введение в язык программирования Go


## История и философия Go


Go — компилируемый, статически типизированный язык программирования с поддержкой конкурентности из коробки. Он был разработан компанией Google в 2007 году не только для улучшения производительности высоконагруженных серверных приложений, но и для продуктивности разработки. И, начиная примерно с 2012 года, язык начал набирать популярность.

Google выдвинула несколько ключевых концепций, которым должен следовать новый язык. Во-первых, он должен быть максимально простым для изучения и быстрого применения. Во-вторых, в его дизайне сознательно отказались от классического ООП (кому-то это может не понравиться) с наследованием и иерархиями классов в пользу более простых механизмов — структур и интерфейсов (с которыми мы познакомимся позже в курсе). Также отцы-основатели языка: Роберт Гризмер (Robert Griesemer), Роб Пайк (Rob Pike) и Кен Томпсон (Ken Thompson), — имеющие громадный опыт программирования систем (Unix / Plan 9) и компиляторов на C++, решили внести в Go более удачные концепции из прочих ЯП:

- быструю компиляцию;
- кросс-платформенность — возможность компиляции под разные ОС и архитектуры из одного исходного кода;
- богатый встроенный инструментарий в виде стандартной библиотеки (в том числе для работы с сетью и тестированием);
- автоматическое управление памятью за счёт сборщика мусора;
- конкурентность — как часть языка.

Поэтому к преимуществам языка мы можем отнести следующие пункты:

- низкий порог входа;
- статическую типизацию;
- простоту языка и минималистичный синтаксис;
- богатую внутреннюю библиотеку;
- реализацию многопоточности на уровне самого языка через его собственные абстракции — горутины;
- самодостаточность и кросс-платформенность;
- автоматическую работу с памятью.

Конечно, у любого языка есть недостатки. И Golang здесь не исключение. К недостаткам (не всегда объективным, стоит сказать) мы можем отнести следующие:

- отсутствие ООП;
- намеренный отказ разработчиков языка от распространённых концепций других языков (дженерики появились пару лет назад);
- нишевость языка, который отлично подходит для бэкенда, сетевых и облачных сервисов, но не для других областей разработки;
- статическая типизация (да, это иногда и минус).

Почему и в преимуществах, и в недостатках языка указана статическая типизация? Потому что это стороны одной очень большой монеты. Статическая типизация выбрана в Go не просто так: она позволяет выявлять ошибки на стадии компиляции, что помогает экономить на сломанных релизах и отладке. Но если говорить о минусах, то тут стоит упомянуть, что порой динамическая типизация позволяет разрабатывать быстрее, создавая более вариативный и лаконичный код.

Среди популярных инструментов, реализованных на языке Go, можно выделить Docker, Kubernetes, крупные инструменты для работы с метриками — такие как Prometheus и Grafana. Golang используют гиганты индустрии. Тут на ум сразу приходит Uber. Более того, весь бэкенд мессенджера TiMe, используемого в ЦУ, является форком проекта Mattermost, который также написан на Go.

## Первая программа Hello World


У нас должно быть всё готово, поэтому приступаем к написанию первой программы. Создаём новый файл с названием `hello_world.go` — в него добавляем указанный ниже код, который будет печатать приветствие.

```go
package main

import "fmt"

/*
	Многострочный
	комментарий
*/

func main() {
	fmt.Println("Hello, world!") // Выводим в консоль
}
```

Теперь разберём, из чего состоит код:

- `package` — это пакет, базовая единица любого сервиса на Golang. Пакеты являются основным способом организации кодовой базы в проектах и зачастую разделение происходит по предоставляемому ими функционалу. `Package main` существует в любой программе, написанной на Go, если она является не библиотекой (о чём чуть позже). Пакет ограничивает видимость неэкспортируемых переменных и функций своими границами. Пакет с объявлением main является точкой входа в твою программу. Соответственно, остальные пакеты проекта, если их необходимо использовать в работе программы, должны быть связаны с входной точкой.
- `import`, как и во многих других языках, позволяет включить код из одного пакета в другой пакет. В нашем случае мы используем импорт пакета `fmt` из стандартной библиотеки, предназначенного для форматирования данных.
- `func main() {...}` реализует вызов функции *main*, являющейся той самой функцией, с которой начинается выполнение любой программы. Без неё ты не запустишь приложение. Заметь, что, вызывая функцию `Println` из пакета `fmt`, мы заключаем весь выполняемый код внутри *main* в фигурные скобки, а строки кода ничем не заканчиваются. (Например, в Go не используется `;` в качестве маркера конца строки. Вместо этого Go на этапе компиляции делает это за нас.)
- Комментарии в Golang можно оставлять двух видов — однострочные и многострочные.
 
Что ещё важно запомнить, так это то, что мы можем писать не только программы, но и библиотеки — а это будут совершенно разные кодовые базы. В программах, сервисах, приложениях абсолютно всегда существует пакет `main` с одноимённой функцией, являющейся точкой входа для запуска описанной логики. Библиотеки же — это набор пакетов, объединённых одной логикой и предоставляющих конкретный функционал, который можно импортировать в программы или прочие библиотеки. Библиотеки не имеют пакета main и создаются только для реализации конкретной цели (обработка строк, преобразование данных из одного формата в другой и прочее). Идеальный пример библиотек — исходный код Golang.
 

## Сборка и кросс-платформенная разработка


Написать программу мало (даже такую простую, как Hello World). Её необходимо скомпилировать, чтобы убедиться в отсутствии ошибок компиляции, и запустить. Для получения бинарного исполняемого файла мы можем использовать команду:

```bash
go build hello_world.go
```

После её выполнения ты получишь исполняемый файл, предназначенный для той ОС и архитектуры, в которой выполняется сама команда. Например, для Windows мы получим файл hello_world.exe. Запустив этот файл, ты увидишь в консоли результат его выполнения.

Но ранее мы говорили о том, что язык позволяет реализовывать лёгкую и удобную кросс-платформенность. Большинство современных языков поддерживают кросс-платформенную разработку, но работать с этими средствами часто непросто. В Go этого можно достичь без особых трудозатрат, добавив к команде `build` необходимые переменные окружения:

- `GOOS` — целевую ОС, под которую планируется создать исполняемый файл;
- `GOARCH` — ожидаемую архитектуру этой системы.

```bash
GOOS=windows GOARCH=amd64 go build hello_world.go
```

В случае если переменные не указываются, то компилятор сам определяет параметры системы, в которой работает.

Однако в Go есть более быстрый способ запуска программ:

```bash
go run hello_world.go
```

Эта команда под капотом вызывает `go build`, компилирует код в исполняемый файл, вызывает его, а затем удаляет по завершении работы. Очень удобно!


## Форматирование


Основная прелесть Golang состоит в том, что довольно много функционала он предоставляет из коробки. Однако, когда мы будем писать код, поймём, что язык очень требователен к соблюдению стандарта форматирования кода. Подобная строгость позволяет создавать читаемый и качественный код. Поэтому Go предоставляет встроенный инструмент для автоматического форматирования кода:

```bash
go fmt ./...
```

При выполнении команды отступы, пробелы и размещение строк приводятся к принятому стилю в языке. Также убираются лишние пробелы и форматируются выражения внутри кодовой базы.


## Переменные окружения


В инструментарии Golang используются переменные окружения — динамические пары в виде ключа и значения к нему, которые хранятся в операционной системе для настройки поведения программ. Go как раз и использует их для настройки своей работы. Для того, чтобы проверить состояние ваших переменных окружения, относящихся к языку, выполните следующую команду:

```bash
go env
```

В терминале вы увидите перечень переменных и их текущих значений. В будущем это знание поможет нам в настройке работы языка.


## Переменные и основные типы данных

### Базовые типы данных в Go


Golang, как и все прочие языки, имеет в своём арсенале перечень основных типов данных, которыми мы будем оперировать на протяжении всего семестра. К ним относятся:

- целые числа;
- числа с плавающей точкой (иногда сюда приписывают и комплексные числа);
- логический тип;
- строки.

Кратко разберём каждый из них.

#### Целые числа

Эти числа могут иметь знак (быть отрицательными), а могут не иметь знака — `int` и `uint` соответственно (смотри таблицу ниже). Для этих чисел важно помнить, какая разрядность (8, 16, 32, 64 бита) в них используется. Если ты видишь, что переменная имеет тип `int8`, то она имеет целочисленный тип со знаком и ограниченной разрядностью в 8 бит. Если же ты видишь переменную с целочисленным типом без разрядности (например, `uint`), то компилятор во время компиляции сам присвоит разрядность этому типу по архитектуре системы, в которой происходит компиляция (32-bit/64-bit).

| **Тип** | **Описание** | **Примеры значений** |
|----|----|----|
| `int` | Целое число (размер зависит от архитектуры: 32 или 64 бита) | `42`, `-10` |
| `int32` | Целое число от −2 147 483 648 до 2 147 483 647 (32 бита) | `-100000`, `2147483647` |
| `int64` | Целое число от −9 223 372 036 854 775 808 до 9 223 372 036 854 775 807 (64 бита) | `-5000000000`, `9223372036` |
| `uint` | Беззнаковое целое число (размер зависит от архитектуры: 32 или 64 бита) | `0`, `100256` |
| `uint32` | Беззнаковое целое число от 0 до 4 294 967 295 | `0`, `42` |
| `uint64` | Беззнаковое целое число от 0 до 18 446 744 073 709 551 615 | `0`, `101` |

#### Числа с плавающей точкой

Для реализации работы с числами, имеющими дробную часть, используются два типа: `float32` (6–7 значащих цифр) и `float64` (15–17 значащих цифр). Первый тип используется в случае, когда точность нам не очень критична, второй тип рекомендуется для большинства вычислений и обеспечивает их высокую точность. И `float32`, и `float64` регулируются стандартом IEEE 754.

**Важный дисклеймер**: эти типы не подходят для проведения финансовых операций и хранения данных о финансах. Как думаешь почему?

| **Тип** | **Описание** | **Примеры значений** |
|----|----|----|
| `float32` | Число с плавающей запятой одинарной точности (32 бита) | `3.14`, `-0.0001` |
| `float64` | Число с плавающей запятой двойной точности (64 бита) | `3.141592653589793`, `0.1` |

#### Логический тип

Логический тип в Golang называется `bool`. Это 1 байт (хотя спецификацией языка не гарантируется, что во всех версиях так и останется), имеющий значение `true` или `false`. Примечательно, что `bool` строго отделён от числовых типов и попытки использовать `1` и `0` вместо `true`/`false` приведут к ошибкам компиляции.

#### Строки

`String` — наиболее важный простой тип в Go, поэтому на нём остановимся поподробнее.

| **Тип** | **Описание** | **Примеры значений** |
|----|----|----|
| `string` | Строка текста | `"Hello, Go!"`, `""` |

Под капотом языка строка — это последовательность байтов. При этом напрямую не предъявляется требование о том, чтобы эти байты были символами какой-то конкретной кодировки, однако всё-таки ожидается, что строка содержит UTF-8 байты, представляющие Unicode-кодовые точки. Ввиду природы строк, по ним можно пройтись циклом:

```go
for i, ch := range "hello" {
	fmt.Println("Номер: ", i, ", символ: ", ch)
}
```

Но здесь ты сразу столкнёшься с тем, что символов алфавита в выводе не увидишь. Вместо них в консоли выведутся числа, кодирующие алфавитные символы в Unicode (101 — это код латинской буквы `e`). И тут стоит быть очень внимательным. Потому что если мы заменим латинскую `e` на кириллическую, то получим совершенно другой результат:

```
Номер:  0 , символ:  104
Номер:  1 , символ:  1077 (кириллица)
Номер:  3 , символ:  108 // Заметь, что номер индекса тут не 2!
Номер:  4 , символ:  108
Номер:  5 , символ:  111

вместо

Номер:  0 , символ:  104
Номер:  1 , символ:  101 (латиница)
Номер:  2 , символ:  108
Номер:  3 , символ:  108
Номер:  4 , символ:  111
```

Поэтому, обходя строки циклом (о котором мы поговорим ниже), мы получаем каждый раз конкретный символ (его кодировку).
Кстати, раз строка — это массив байтов, то мы можем обращаться к её элементам по индексу. Но в этом случае у нас будет доступ не к символу / его представлению в кодировке, а к конкретному байту. **Будь внимательнее!**

Но это не все нюансы строк. Чтобы узнать *количество байтов* в строке, надо воспользоваться встроенной функцией `len()`:

```go
package main

import "fmt"

func main() {
	fmt.Println(len("hello"))
	fmt.Println(len("привет"))
}
```

Мы будем ожидать, что ответ будет 5 и 6 байт. Но на самом деле получим 5 и 12 байт в ответе. Почему? Потому что каждый символ кириллицы кодируется двумя байтами (поэтому в примере выше индекс «поехал») и `len()` честно нам об этом говорит. Предлагаю тебе самостоятельно поискать способ подсчёта именно букв в любой строке :)

И **самое важное**: ***строки в Go неизменяемы!*** Это важно помнить. Ты не можешь через обращение по индексу изменить символ строки. При конкатенации (объединении строк) ты не прибавляешь к одной строке вторую — ты создаёшь новую строку.

```go
s1 := "Строка "
s2 := "и еще строка"
s1 = s1 + s2
fmt.Println(s1)
```

В этом примере ты получишь ожидаемый ответ, но для нового значения выделится новая область памяти, с которой будет работать переменная `s1`.

Интересно вот еще что: в примере выше мы использовали конкатенацию при объединении строк. Но в пакете (структурная единица проектов в Go — мы к ним еще обязательно вернемся) стандартной библиотеки `strings` вы можете найти некий `Builder`, который позволяет ускорить данную операцию, если мы проделываем ее постоянно. Также в этом пакете ты найдёшь множество других полезных функций для работы со строками. Ссылка в самом низу лонгрида, так что советуем ознакомиться с документацией.

#### Более хитрые типы

В Golang существуют типы-синонимы/алиасы, которые являются лишь синтаксической обёрткой рассмотренных нами типов:

| **Тип** | **Описание** | **Примеры значений** |
|----|----|----|
| `byte` | Синоним типа `uint8`, представляет байты | `'a'`, `97` |
| `rune` | Синоним типа `int32`, представляет символ в Unicode | `'ы'`, `0x0000044B` |

Роль байтов, наверное, понятна. Но что такое *руны*?

Это алиас на представление символов в Unicode, а значит, когда мы проходимся циклом по строке, то получаем как раз руны, с которыми уже можно взаимодействовать как с отдельными символами, так как при обходе строки "привет" мы получим именно символы, так как Go под капотом склеивает байты в единый символ (напомню, что кириллица кодируется двумя байтами).


### Способы определения переменных


Есть несколько способов определения переменных. Классическое полное определение переменной выглядит так:

```go
var i int = 1
var w float64 = 12.5
```

Здесь видны все части определения переменной: ключевое слово var, имя, тип и значение. В такой записи указание типа требуется только для явного указания типа переменной, если оно не выводится автоматически из значения (например, переменная должна хранить явный `int64`). Поэтому зачастую указание типа отбрасывают из ненадобности:

```go
var i = 1         // int
var w = 12.5      // float64
var name = "John" // string
```

Если значение отсутствует, то переменная объявляется с «нулевым» для данного типа значением: `0` для чисел, `false` для булевого типа, пустая строка для строк:

```go
var i int          // 0
var w float64      // 0.0
var isPresent bool // false
var name string    // ""
```

Иначе можно сказать, что в Golang переменная, объявленная без явной инициализации, автоматически получает `zero value`, соответствующее её типу. Подобный подход исключает наличие неинициализированной памяти и позволяет безопасно использовать переменные сразу после их объявления.

Можно определять несколько переменных сразу единым блоком:

```go
var i, j, k int = 1, 2, 3
var (
	name       string = "John Doe"
	occupation string = "gardener"
)
```

И наконец, короткое определение переменной с помощью оператора `:=`:

```go
name := "John Doe"
age := 34
```

Обрати внимание, что в Go при коротком объявлении переменной используется именно `:=`, а не `=`, как в Python.


### Преобразование типов


В Go нет неявных преобразований между разными типами. Поэтому нужно указывать целевой тип явно:

```go
var i = 1
var v float64 = float64(i)
```


### Константы


Отдельно стоит упомянуть константы. Это неизменяемые переменные, которые определяются на стадии компиляции. Они объявляются с помощью ключевого слова `const`:

```go
const Pi = 3.14159

const name string = "My name"
const surname = "My surname"

const (
	StatusOK       = 200
	StatusNotFound = 404
)
```

При объявлении мы можем использовать конкретный тип для константы. А можем доверить Go выбор её типа, если он нам не особо важен.


### Области видимости и экспортируемость


*Область видимости* — это участок кода, в котором переменная, функция, константа и прочие элементы остаются доступными для использования. Область видимости констант и переменных внутри функции ограничивается телом самой функции (то есть фигурными скобками - `{}`).

Экспортируемость определяется регистром первого символа, с которого начинается название переменной, функции и так далее. Если именование начинается с большой буквы, то элемент кода экспортируемый и может использоваться в разных местах проекта; если же именование начинается с маленькой буквы, то увы — элемент можно использовать только в рамках видимости пакета (пока нам важно помнить, что пакет является единицей разделения кода на самодостаточные элементы).


## Управляющие конструкции

### if/else


Оператор ветвления `if/else` позволяет выполнять код по достижении определённых условий, чем добавляет вариативности в наш код. Например, если нас просят купить хлеб, а если будет и колбаса, то взять и колбасу, то с помощью ветвления мы можем написать следующее:

```
покупки := func КупитьХлеб()

if есть колбаса {
	покупки + колбаса
}
```

Простой живой пример:

```go
x := 10

if x > 5 {
	fmt.Println("x больше 5") // Текст выведется в консоль
}
```

Важные моменты работы с этой управляющей конструкцией:
- условие у `if` должно быть выражением типа `bool` (так как в Go нет неявных преобразований);
- условие не требуется оборачивать круглыми скобками (в отличие от многих C-подобных языков);
- переменные, объявленные внутри тела блока, видны только внутри этого блока;
- переменные, объявленные внутри условия операторов (об этом ниже), видны во всей управляющей конструкции, но не за её пределами.

Очень полезная особенность Go — возможность объявления и инициализации переменной прямо в условии оператора ветвления, с чем мы будем время от времени работать:

```go
if x := getValue(); x > 0 {
	fmt.Println("Положительное значение:", x)
}
// x здесь уже недоступна
```

Это дает нам возможность дополнительно ограничивать области видимости тех вещей, которые нужны лишь в конкретном месте кода. А если нам требуется больше, чем одно условие, то мы можем использовать конструкцию `else-if`. Вот так она выглядит:

```go
if x > 0 {
	fmt.Println("Положительное")
} else if x < 0 {
	fmt.Println("Отрицательное")
} else {
	fmt.Println("Ноль")
}
```

Как водится, мы можем использовать несколько `else-if` в одной конструкции без использования чистого `else`. Или только `else`, без промежуточных вариантов ветвления. Но в первом случае мы можем построить очень громоздкие конструкции, что почти всегда неудобно. Посмотри:

```go
if x == 1 {
	...
} else if x == 2 {
	...
} else if x == 3 {
	...
} else if x == 4 {
	...
} else if x == 5 {
	...
} else if x == 6 {
	...
}
```

Напоминает очень неоптимальный код, который довольно сложно читать. Но порой без подобной логики никуда, и тогда используется другая конструкция.


### switch


`Switch` как раз и используется для случаев, когда нам надо выбрать один из множества вариантов развития событий. Конструкция делает подобные случаи очень читаемыми. Общая форма `switch` выглядит так:

```go
switch time.Now().Weekday() {
case time.Saturday:
	fmt.Println("Today is Saturday")
case time.Sunday:
	fmt.Println("Today is Sunday")
default:
	fmt.Println("Today is a working day")
}
```

Здесь важно понять, как вообще работает вся конструкция:

- `switch` содержит в себе некоторое значение, с которым мы будем работать во всех условиях выбора. Например, в нашем случае `switch` может иметь значение понедельника `time.Monday`.
- `case` — это наши варианты выбора. Кейсы. Программа идёт по ним последовательно сверху вниз, и если бы мы нашли `case` со значением `time.Monday`, то выполнили бы код этого кейса и сразу вышли бы из конструкции, не проверяя дальнейшие условия.
- `default` позволяет нам перехватывать все остальные случаи, которые мы не рассмотрели. Как раз в нашем примере эта часть `switch` и сработает. Кстати, можно обходиться и без `default`, но его использование является правилом хорошего тона, а также защитой нас, как разработчиков, от возможных неучтённых сценариев.

Но можно не использовать никакого значения у самого `switch`. Он сам по себе может быть пустым. И если так происходит, то это эквивалентно простому `switch true`. Это позволяет использовать в `case` не значения, а условия:

```go
hour := time.Now().Hour()
switch {
case hour < 12:
	fmt.Println("Good morning!")
case hour < 17:
	fmt.Println("Good afternoon!")
default:
	fmt.Println("Good evening!")
}
```

Выполнением `switch` в Go можно управлять двумя операторами: `break` и `fallthrough`. Первый позволяет выйти из `switch` раньше, чем нам обычно нужно (например, в случае корнер-кейса). Оператор `fallthrough` выполняет следующую ветку `case` без повторной проверки её условия. Он должен находиться в конце блока кода внутри своего `case`. И после того как целевой `case` завершит свою логику, то выполнение программы продолжится в `case`, находящемся прямо следом. Так можно собирать цепочки выполнения кейсов в случае, если они друг от друга зависят.


## Множественные выражения


Иногда нам необходимо проверять за раз несколько условий. И тогда нам на помощь приходят логические операторы.

**Оператор** `&&` (И). Он возвращает `true`, если оба значения истинны, как в нашем примере:

```go
isUser := true
isAdmin := true

if isUser && isAdmin {
	fmt.Println("Operation completed") // Произойдёт печать в консоль
}
```

**Оператор** `||` (ИЛИ). Он возвращает `true`, если хотя бы одно из значений истинно:

```go
isClosed := true
isWeekend := false

if isClosed || isWeekend {
	fmt.Println("You can't order now") // Тоже увидим текст в консоли
}
```

**Оператор** `!` (НЕ). Возвращает `true`, если исходное выражение `false`, и наоборот. Пример:

```go
isClosed := true
isWeekend := true

if !(isClosed || isWeekend) {
	fmt.Println("You can order now") // Ничего не выведем, потому что рабочий день :С
}
```


## Циклы


Иногда нужно выполнить один и тот же фрагмент кода несколько раз. Копировать его каждый раз не лучшая идея. Для этой задачи используют операторы циклов.
В отличие от многих C-подобных языков программирования, в Go применение оператора `for` — единственный способ организации циклов. Оператор `while` в Go идеологически отсутствует. А полная функциональность языка обеспечивается за счёт четырёх разновидностей оператора `for`:
- полной формы оператора `for` в стиле C;
- оператора `for`, использующего только условие;
- бесконечной формы оператора `for`;
- оператора `for-range`.

Синтаксис полной формы оператора `for` выглядит следующим образом (обычный цикл с инкрементом):

```go
sum := 0

for i := 1; i < 5; i++ {
	sum += i
}

fmt.Println(sum) // 10
```

То есть мы с помощью одной переменной задаём начальную точку, условие окончания цикла и шаг инкремента. Заметь ещё раз, что переменная `i` видна только внутри цикла и условия его задания.

Одной из разновидностей цикла `for` является вариант, в котором опущена инициализация и шаг инкремента, а используется только условие. Это аналог цикла `while` в других языках программирования:

```go
n := 1

for n < 5 {
	n *= 2
}

fmt.Println(n) // 8
```

Бесконечный цикл получится, если опустить все три части заголовка `for`:

```go
sum := 0

for {
	sum++
	fmt.Println(sum) // Будем «бесконечно» писать числа в консоль
}
```

Для того чтобы взаимодействовать с различными структурами данных (помнишь, раньше мы работали со string?), используется конструкция `for-range`. Она используется по полному итерированию не только по строкам, но и массивам, мапам и так далее (эти структуры данных — темы следующих уроков). Базовый синтаксис этого варианта цикла выглядит так:

```go
for индекс, значение := range структура данных {
// Тело цикла
}
```

Или как в нашем прошлом примере со строками:

```go
for i, ch := range "hello" {
	fmt.Println("Индекс: ", i, ", значение: ", ch)
}
```

Забавно, что итерирование по целым числам изначально в Go не было реализовано, а появилось только несколько лет назад в версии Go 1.22:

```go
for i := range 5 {
	fmt.Println(i) // 0, 1, 2, 3, 4
}
```

И это является аналогом:

```go
for i := 0; i < 5; i++ {
	fmt.Println(i)
}
```

Однако работа с циклами на этом не ограничивается. Иногда приходится писать цикл в цикле (старайся избегать этой практики и писать более оптимизированный код). В случае когда требуется прервать выполнение всех родительских циклов из вложенного, используются так называемые `метки` — строковый лейбл, который позволяет через своё название прервать выполнение кода за счёт обращения к нему через оператор `break`, который в обычном своём состоянии может помочь выйти только из одного цикла за раз:

```go
sum := 0

outer:
for {
	sum++
	for j := 0; j < sum; j++ {
		if j*sum == 111 {
			break outer // Тут мы выйдем из всех циклов разом. Если бы этот код не имел метки/лейбла, то мы бы просто прекратили итерацию вложенного цикла и вернулись бы во внешний бесконечный цикл
		}
	}
}
```

Но бывают и ситуации, когда работа в рамках текущей итерации уже завершена и нужно досрочно перейти на следующую итерацию. Для этого существует оператор `continue`:

```go
sum := 0

for i := 1; i < 5; i++ {
	if i%2 != 0 {
		continue // Пропустить нечётные
	}

	sum += i
}

fmt.Println(sum) // 6 (2+4)
```

Если требуется итерироваться только по индексам, второе значение можно опустить:

```go
strings := [3]string{"I", "love", "Go"} // Массив, познакомимся с ним ближе в следующий раз
for i := range strings {
	fmt.Println(strings[i])
}
```

А чтобы итерироваться только по значению, опускаем первое значение (индекс) с помощью символа подчёркивания (`_`):

```go
strings := []string{"I", "love", "Go"}
for _, s := range strings {
	fmt.Println(s)
}
```
